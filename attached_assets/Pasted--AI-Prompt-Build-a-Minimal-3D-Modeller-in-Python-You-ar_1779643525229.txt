# AI Prompt — Build a Minimal 3D Modeller in Python

You are an expert graphics programmer, software architect, and Python educator specializing in OpenGL, computational geometry, CAD systems, and clean software design.

Your task is to design and implement a compact but extensible 3D modeller in Python inspired by classic CAD architecture and scene graph systems.

## Core Objective

Build a lightweight interactive 3D modeller in approximately 500–1000 lines of clean, well-structured Python code.

The system must demonstrate the foundational architecture behind professional CAD and 3D modelling software while remaining educational, readable, and modular.

---

# Technical Stack

Use:

* Python 3
* PyOpenGL
* GLUT (OpenGL Utility Toolkit)
* NumPy

Prefer Legacy OpenGL fixed-function pipeline for simplicity and educational clarity.

Do NOT use:

* Game engines
* Modern OpenGL shader pipelines
* Heavy GUI frameworks unless explicitly required
* External scene graph libraries

---

# Required Features

Implement the following systems:

## 1. Viewer / Rendering Engine

Create a `Viewer` class responsible for:

* Window creation
* OpenGL initialization
* Main render loop
* Projection matrix setup
* Camera/view transformations
* Buffer clearing and rendering passes
* Interaction integration

The renderer should:

* Enable depth testing
* Support lighting
* Use matrix stack operations (`glPushMatrix`, `glPopMatrix`)
* Render a simple grid plane
* Maintain model-view matrices

---

## 2. Scene Graph Architecture

Implement a hierarchical scene graph using the Composite Design Pattern.

Required classes:

* `Scene`
* `Node` (abstract base class)
* `Primitive`
* `Cube`
* `Sphere`
* `HierarchicalNode`
* `SnowFigure` (example composite object)

Each node should support:

* Translation
* Scaling
* Selection state
* Rendering
* Bounding boxes (AABB)

The scene should:

* Maintain node collections
* Handle selection
* Dispatch manipulation commands

---

## 3. Coordinate Spaces & Transformations

Correctly implement:

* Model space
* World space
* View space
* Projection transformations

Use:

* Translation matrices
* Scaling matrices
* Matrix multiplication
* Inverse model-view matrices

Explain transformation flow clearly in comments.

---

## 4. Interaction System

Build an event-driven interaction system using GLUT callbacks.

Support:

* Mouse movement
* Mouse clicks
* Keyboard input
* Scroll wheel zoom
* Scene rotation
* Scene translation

Implement:

* Callback registration system
* Event dispatching
* Application-level interaction abstraction

Avoid tightly coupling input handling to rendering logic.

---

## 5. Trackball Camera Rotation

Implement or simulate a trackball-style camera controller.

Requirements:

* Smooth intuitive scene rotation
* Quaternion-safe logic if possible
* Mouse drag rotation

Explain:

* Why trackballs are used
* How they avoid gimbal lock

---

## 6. Object Picking

Implement ray-based object selection.

Requirements:

* Generate rays from mouse coordinates
* Use projection unprojection (`gluUnProject`)
* Ray-AABB intersection tests
* Closest-hit selection logic

Discuss:

* Accuracy vs performance tradeoffs
* Why AABB approximation is used

---

## 7. Object Manipulation

Support:

* Moving selected objects
* Scaling selected objects
* Color cycling
* Adding primitives dynamically

Objects should update:

* Transformation matrices
* Bounding boxes
* Scene state

---
